1.1 Latar Belakang

Politeknik Negeri Manado sebagai salah satu perguruan tinggi vokasi di Indonesia memiliki peran penting dalam menghasilkan lulusan yang kompeten dan siap kerja di bidang masing-masing. Program Studi D4 Teknik Listrik merupakan salah satu program studi di bawah naungan Politeknik Negeri Manado yang secara aktif menyelenggarakan kegiatan akademik meliputi proses perkuliahan, penelitian, pengabdian kepada masyarakat, dan berbagai kegiatan kemahasiswaan. Dalam menjalankan seluruh aktivitas tersebut, program studi memiliki kebutuhan untuk menyajikan informasi yang akurat, lengkap, dan terkini kepada berbagai pihak termasuk mahasiswa aktif, calon mahasiswa, dosen, tenaga kependidikan, serta masyarakat umum.

Seiring dengan perkembangan teknologi informasi dan komunikasi yang semakin pesat, kebutuhan akan akses informasi secara digital menjadi hal yang tidak dapat dihindari. Masyarakat saat ini terbiasa mencari informasi melalui internet sebelum mengambil keputusan, termasuk dalam memilih program studi untuk melanjutkan pendidikan. Calon mahasiswa yang ingin mengetahui informasi mengenai kurikulum, profil dosen, fasilitas laboratorium, dan kegiatan program studi umumnya akan mencari informasi tersebut melalui website resmi. Oleh karena itu, ketersediaan website program studi yang informatif dan profesional menjadi salah satu faktor penting dalam menarik minat calon mahasiswa.

Namun pada kenyataannya, pengelolaan dan penyebaran informasi Program Studi D4 Teknik Listrik masih dilakukan secara konvensional dan belum terintegrasi dalam satu platform digital yang terpusat. Data mengenai profil dosen beserta riwayat akademiknya, struktur kurikulum dan daftar mata kuliah, karya ilmiah dosen yang mencakup publikasi, penelitian, pengabdian, buku ajar, HKI, dan sertifikasi, dokumentasi galeri kegiatan tridharma, serta informasi fasilitas laboratorium masih tersebar di berbagai dokumen fisik maupun file digital yang tidak saling terhubung. Kondisi ini menyebabkan informasi sulit diakses oleh pihak yang membutuhkan dan rentan terhadap ketidaksesuaian data karena pembaruan yang tidak terpusat.

Permasalahan lain yang muncul adalah terkait pengelolaan konten informasi program studi. Saat ini, proses pembaruan data masih bergantung pada satu atau dua orang yang memiliki kemampuan teknis tertentu. Tidak adanya sistem manajemen konten yang memungkinkan pembagian tugas berdasarkan peran pengguna menyebabkan proses pengelolaan data menjadi tidak efisien. Sebagai contoh, dosen yang ingin memperbarui profil atau menambahkan karya ilmiah terbaru harus menyampaikan data tersebut kepada pengelola secara manual, kemudian pengelola yang akan memasukkan data ke dalam sistem. Proses seperti ini memakan waktu, rawan kesalahan input, dan membuat data yang ditampilkan seringkali tidak terkini.

Selain itu, tidak adanya pembagian hak akses yang jelas juga menimbulkan risiko keamanan data. Semua pengelola memiliki akses yang sama terhadap seluruh data tanpa adanya pembatasan berdasarkan peran dan tanggung jawab masing-masing. Idealnya, seorang admin memiliki akses penuh untuk mengelola seluruh data, seorang dosen hanya dapat mengelola data profil dan karya ilmiahnya sendiri, dan seorang pegawai hanya dapat mengelola data yang menjadi tanggung jawabnya seperti kurikulum dan fasilitas. Pembagian hak akses seperti ini penting untuk menjaga integritas dan keamanan data program studi.

Di sisi teknologi, perkembangan framework dan tools pengembangan web telah mengalami kemajuan yang sangat signifikan dalam beberapa tahun terakhir. Teknologi seperti Next.js yang merupakan framework berbasis React telah menjadi standar industri dalam pengembangan aplikasi web modern. Next.js menawarkan berbagai keunggulan seperti server-side rendering untuk performa yang lebih baik, routing berbasis file system yang memudahkan pengembangan, dan optimasi otomatis untuk gambar dan font. Ditambah dengan TypeScript yang memberikan keamanan tipe data pada saat pengembangan, Tailwind CSS yang mempercepat proses styling dengan pendekatan utility-first, serta Supabase sebagai layanan backend-as-a-service yang menyediakan database PostgreSQL, autentikasi pengguna, dan penyimpanan file secara terintegrasi, maka tersedia ekosistem teknologi yang sangat mendukung untuk membangun aplikasi web yang modern, responsif, dan aman.

Di era transformasi digital saat ini, keberadaan sistem informasi berbasis web menjadi kebutuhan yang mendesak bagi setiap institusi pendidikan. Website program studi tidak hanya berfungsi sebagai media penyampaian informasi kepada publik, tetapi juga berperan sebagai sarana promosi untuk menarik minat calon mahasiswa, wadah dokumentasi kegiatan akademik dan non-akademik, media penampilan karya ilmiah dan prestasi dosen, serta alat bantu pengelolaan data akademik secara terpusat dan efisien. Dengan adanya sistem informasi berbasis web yang terintegrasi, program studi dapat meningkatkan profesionalisme dalam penyajian informasi, memperluas jangkauan promosi ke tingkat nasional maupun internasional, serta mempermudah koordinasi antar civitas akademika dalam pengelolaan data.

Berdasarkan permasalahan dan kebutuhan yang telah diuraikan, maka dalam mata kuliah Teknologi Web ini dikembangkan sebuah Sistem Informasi Program Studi D4 Teknik Listrik berbasis web. Sistem ini dirancang untuk menjawab tiga permasalahan utama yaitu penyajian informasi akademik kepada publik, pengelolaan konten dengan pembagian hak akses berdasarkan peran pengguna, serta penerapan teknologi web modern dalam pembangunan aplikasi. Sistem ini dibangun menggunakan Next.js sebagai framework utama, React untuk komponen antarmuka pengguna, TypeScript untuk keamanan tipe data, Tailwind CSS untuk desain yang responsif, dan Supabase sebagai layanan backend yang mencakup database PostgreSQL, autentikasi pengguna, dan penyimpanan file.


1.2 Rumusan Masalah

Berdasarkan latar belakang yang telah diuraikan, maka rumusan masalah dalam proyek ini adalah sebagai berikut.

1. Bagaimana membangun sistem informasi program studi berbasis web yang dapat menyajikan informasi akademik secara lengkap dan terkini kepada publik.

2. Bagaimana merancang sistem manajemen konten dengan pembagian hak akses berdasarkan peran pengguna sehingga pengelolaan data dapat dilakukan secara efisien oleh admin, dosen, dan pegawai.

3. Bagaimana mengimplementasikan teknologi web modern untuk menghasilkan aplikasi yang responsif, cepat, dan mudah digunakan.


1.3 Tujuan Proyek

Berdasarkan rumusan masalah yang telah diuraikan, maka tujuan dari proyek ini adalah sebagai berikut.

1. Membangun sistem informasi program studi berbasis web yang menyajikan informasi akademik secara lengkap dan terkini kepada publik, meliputi profil program studi, data dosen beserta karya ilmiahnya, struktur kurikulum dan daftar mata kuliah, galeri kegiatan tridharma, fasilitas laboratorium, serta berita terkini.

2. Merancang dan mengimplementasikan sistem manajemen konten dengan pembagian hak akses berdasarkan tiga peran pengguna yaitu admin yang memiliki akses penuh untuk mengelola seluruh data, dosen yang dapat mengelola profil dan karya ilmiah secara mandiri, serta pegawai yang dapat mengelola kurikulum, fasilitas, dan konfigurasi website.

3. Mengimplementasikan teknologi web modern meliputi Next.js sebagai framework utama, React untuk komponen antarmuka, TypeScript untuk keamanan tipe data, Tailwind CSS untuk desain responsif, dan Supabase untuk layanan database, autentikasi, dan penyimpanan file sehingga menghasilkan aplikasi yang responsif, cepat, dan mudah digunakan.


1.4 Manfaat Proyek

Berdasarkan tujuan yang ingin dicapai, manfaat yang diharapkan dari proyek ini adalah sebagai berikut.

1.4.1 Manfaat bagi Pengguna

a. Dengan terbangunnya sistem informasi berbasis web, calon mahasiswa dapat mengakses informasi lengkap mengenai program studi, kurikulum, fasilitas, dan profil dosen secara online kapan saja dan di mana saja tanpa harus datang langsung ke kampus.

b. Dengan adanya sistem manajemen konten berbasis peran, dosen dapat mengelola profil dan karya ilmiahnya secara mandiri melalui dashboard tanpa bergantung pada pihak lain, sehingga data yang ditampilkan selalu terkini.

c. Admin dan pegawai program studi dapat mengelola konten website secara efisien sesuai dengan hak akses masing-masing tanpa memerlukan keahlian teknis pemrograman.

d. Masyarakat umum dapat mengetahui informasi terkini mengenai kegiatan dan perkembangan program studi melalui halaman publik yang informatif dan mudah diakses.

1.4.2 Manfaat bagi Kampus

a. Tersedianya website program studi yang profesional dan informatif dapat meningkatkan visibilitas dan citra Program Studi D4 Teknik Listrik Politeknik Negeri Manado di dunia digital.

b. Sistem manajemen konten dengan pembagian peran memungkinkan pengelolaan data akademik secara terpusat dan terorganisir, sehingga mendukung proses akreditasi dan pelaporan program studi.

c. Penerapan teknologi web modern dalam proyek ini dapat menjadi referensi dan contoh implementasi yang dapat diadopsi oleh program studi lain di lingkungan Politeknik Negeri Manado.

d. Website yang responsif dan cepat dapat menjadi sarana promosi yang efektif untuk menarik minat calon mahasiswa dari berbagai daerah.

1.4.3 Manfaat bagi Mahasiswa

a. Menambah pengetahuan dan keterampilan dalam membangun sistem informasi berbasis web yang menyajikan data akademik secara lengkap menggunakan teknologi modern seperti Next.js, React, TypeScript, dan Supabase.

b. Memberikan pengalaman langsung dalam merancang sistem manajemen konten dengan konsep pembagian hak akses berdasarkan peran pengguna yang merupakan kebutuhan umum di dunia industri.

c. Melatih kemampuan dalam mengimplementasikan teknologi web modern yang mencakup frontend, backend, database, autentikasi, dan penyimpanan file dalam satu aplikasi terintegrasi.

d. Meningkatkan pemahaman tentang konsep keamanan aplikasi web melalui implementasi Row Level Security pada database dan role-based access control pada sistem autentikasi.


1.5 Ruang Lingkup dan Batasan Proyek

Ruang lingkup proyek ini mencakup fitur-fitur yang dikembangkan dalam sistem informasi Program Studi D4 Teknik Listrik sebagai berikut.

1. Halaman Publik
   a. Halaman beranda yang menampilkan hero section, sambutan ketua program studi, statistik mahasiswa aktif, grafik data mahasiswa per semester, dan berita terkini.
   b. Halaman tentang yang menyajikan informasi umum mengenai program studi.
   c. Halaman staf dan dosen yang menampilkan profil lengkap dosen beserta karya ilmiah.
   d. Halaman kurikulum yang menampilkan struktur kurikulum, daftar mata kuliah per semester, dan capaian pembelajaran lulusan.
   e. Halaman galeri yang menampilkan dokumentasi kegiatan tridharma dan fasilitas laboratorium.
   f. Halaman fasilitas yang menampilkan sarana dan prasarana program studi.

2. Sistem Autentikasi
   a. Halaman login untuk pengguna terdaftar.
   b. Pembagian hak akses berdasarkan tiga peran yaitu admin, dosen, dan pegawai.

3. Dashboard Admin
   a. Pengelolaan data staf dan dosen meliputi tambah, ubah, dan hapus data.
   b. Pengelolaan karya ilmiah dosen meliputi publikasi, penelitian, pengabdian, buku ajar, HKI, dan sertifikasi.
   c. Pengelolaan galeri dan fasilitas.
   d. Pengelolaan kurikulum dan mata kuliah.
   e. Konfigurasi website.

4. Dashboard Dosen
   a. Pengelolaan profil dosen secara mandiri.
   b. Pengelolaan karya ilmiah milik dosen yang bersangkutan.

5. Dashboard Pegawai
   a. Pengelolaan kurikulum dan fasilitas.
   b. Pengelolaan konfigurasi website.

6. Penyimpanan Data
   a. Database menggunakan Supabase PostgreSQL untuk menyimpan data dosen, karya ilmiah, galeri, kurikulum, mata kuliah, capaian pembelajaran, dan profil pengguna.
   b. Penyimpanan file menggunakan Supabase Storage untuk foto profil dosen, gambar galeri, dan dokumen kurikulum.
   c. Keamanan data menggunakan Row Level Security pada database dan kebijakan akses pada storage bucket.

Batasan proyek ini adalah sebagai berikut.

1. Sistem ini dikembangkan khusus untuk Program Studi D4 Teknik Listrik Politeknik Negeri Manado.
2. Aplikasi berjalan pada platform web dan diakses melalui browser, tidak mencakup pengembangan aplikasi mobile native.
3. Sistem tidak mencakup fitur pendaftaran mahasiswa baru atau sistem akademik seperti pengisian KRS dan nilai.
4. Data awal yang ditampilkan pada website merupakan data contoh untuk keperluan demonstrasi.
5. Proyek ini dikembangkan sebagai tugas mata kuliah Teknologi Web.
